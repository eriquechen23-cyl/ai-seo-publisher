BackgroundTasks + Job Polling Milestone

2026-07-03 21:22 Asia/Taipei · ai-seo-publisher · Codex

本階段將原本同步等待 LLM 與 WordPress 的流程，改為 FastAPI BackgroundTasks 背景處理搭配前端 job polling。目標是回應 Task 4 的架構韌性需求，避免外部 API latency 讓前端卡在單一 request。

目前狀況

POST /api/v1/articles/generate-publish 現在會先建立 SQLite job，立即以 HTTP 202 回傳 job_id、RECEIVED 與 status_url。LLM 生成、內容驗證、修復與 WordPress draft 建立改由背景任務執行。

架構流程

實作摘要

Backend API

新增 enqueue response 與 GET /articles/jobs/{job_id} 狀態查詢。

Workflow

拆成 enqueue() 與 process_job()，背景任務將錯誤落庫。

Frontend

React Query mutation 建立任務，query 每 1.5 秒輪詢直到完成或失敗。

韌性

保留 LLM timeout retry，WordPress 連線錯誤會標記為 retryable job error。

驗證狀態

已執行 backend\.venv\Scripts\python.exe -m pytest backend\tests，7 個測試通過。已執行 npm run build，TypeScript 與 Vite production build 通過。
