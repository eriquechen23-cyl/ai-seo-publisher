Agent Router Rule-Based Skill Report

Codex · 2026-07-04 · AI SEO Publisher

目前狀態

系統原本只要存在 ArticleResearchTool 就會進行 research。 本次新增 ArticleAgentRouter，讓 pipeline 先依規則判斷是否需要 tool， 再決定要呼叫 DuckDuckGo / Brave / mock research，或直接回傳 router-skip context。

Rule-Based Router

auto

依文章 brief 規則判斷是否需要外部搜尋。這是產品化預設方向。

always

每次都呼叫 research tool，適合現場 Demo 或想展示 tool usage。

never

永遠跳過 research tool，適合離線測試或避免外部呼叫。

流程圖

新增檔案與介面

  • backend/app/services/article_agent_router.py：rule-based routing skill。
  • docs/agent-router-skill.md：router skill 文件，描述 input、output、規則與安全邊界。
  • RESEARCH_ROUTER_MODE：支援 auto、always、never。
  • ArticleAgentPipeline：在 research 前先呼叫 router。

驗證

  • 後端測試：18 passed。

  • 新增測試涵蓋 always、never、auto、internal brief skip，以及 pipeline skip research 的路徑。

下一步

短期可把 router decision 顯示在 job metadata，讓前端或審稿頁看到「為什麼有或沒有調用 tool」。 中期可把 rule set 改成設定檔；長期再升級成 LLM router 或多工具 router。
