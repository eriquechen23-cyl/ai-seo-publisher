AI SEO Publisher01

Problem

專案要解決的問題

內容產製耗時

SEO 文章需要搜尋資料、撰寫、檢查、修稿與發布，人工流程容易分散。

生成內容需要審稿

單次 LLM 生成不一定符合 SEO、HTML 結構、CTA 與品質需求。

搜尋資料需要受控

LLM 不應直接瀏覽網路；後端應限制查詢、結果數與注入內容。

發布需要可追蹤

非同步工作必須讓前端知道任務狀態、是否調用工具、最終是否建立草稿。

Search-aware Generation06

Agent Router

Rule-based 決定何時調用工具

auto

根據文章 brief 自動判斷是否需要外部資料。

always

Demo 時固定調用 Search Tool，方便展示流程。

never

離線測試或避免外部請求時使用。

觸發規則包含時效、比較、SEO、AI、WordPress、REST API、SaaS、ESG 與多 keyword brief。

Routing Skill07

Critic Pattern

Producer + Critique 的反思回路

Producer

依 brief 與 research context 產出 WordPress-ready HTML、標題、CTA 與段落結構。

Critique

檢查 SEO、HTML 結構、關鍵字使用、CTA、內容品質與是否需要 revision。

Validator 做 deterministic checks；Critique 做語意品質判斷。兩者都通過才進入 WordPress draft。
